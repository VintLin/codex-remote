## 结论先行

截至 **2026-06-21**，这些能力不应按“协议里有”直接进入 Codex Remote 近期 stage。官方把 `codex` 与 `codex app` 标为 Stable，但把 `codex app-server` 标为 **Experimental**；官方对 Experimental 的定义是“不稳定、可能移除或改变、风险自担”。因此 Codex Remote 应把 app-server 高级能力先投影为 **Worker capability/status**，只有进入明确用户意图时才做 Web UI。([OpenAI Developers][1])

官方 Remote connections 的产品语义是：从另一台设备控制已连接的 Mac/Windows host，能力集中在 **start / steer / approve / review outputs/diffs/tests/terminal/screenshots / host switching**。这正是 Codex Remote 近期主线；voice、feedback、外部配置迁移、Windows sandbox setup 都不是同一层级的主路径。([OpenAI Developers][2])

| **Windows sandbox**：`windowsSandbox/setupStart`、`windowsSandbox/readiness`、`windows/worldWritableWarning` | 官方 Windows 文档说明 Codex 可在 Windows native app/CLI/IDE 中运行；native Windows agent mode 用 Windows sandbox 限制工作目录外写入与未批准网络访问。Windows sandbox 有 `elevated` 与 `unelevated` 两种模式，`elevated` 依赖低权限 sandbox users、文件系统边界、防火墙和本地策略。([OpenAI Developers][5])          | app-server 官方文档只明确 `windowsSandbox/setupStart`：自定义 Windows client 可异步触发 sandbox setup，之后收到 `windowsSandbox/setupCompleted`。`windowsSandbox/readiness` 有公开 issue 显示前后端版本错配时 backend 不认识该 variant；公开 commit 列表显示后来有 “Add Windows sandbox readiness RPC”。`windows/worldWritableWarning` 只看到公开流水线/提交标题层面的 “world-writable warning improvements”，证据不足以产品化。([OpenAI Developers][6]) | **later**；其中 `readiness/worldWritableWarning` 属于 **experimental-watch / insufficient evidence**。 | 只在 Windows device detail 或 status banner 显示：`sandbox: ready / needs setup / degraded / unsupported`。不要做全局入口，不要暴露 raw setup button。       | Worker-only。setup/readiness 必须发生在 Windows host；可能需要管理员权限、本地策略、防火墙规则、WSL/native 环境判断。Control Plane/Web 不应直接执行 setup，只发用户明确确认后的 Worker command。 |

| **Rate limits / usage**：`account/rateLimits/read`、`account/usage/read`                                    | Codex App settings 的 Profile 已有 activity insights、lifetime tokens、peak tokens、streaks、longest task、token activity。([OpenAI Developers][8])                                                                                                              | app-server 官方文档支持 `account/rateLimits/read`，返回 used percent、window、reset time、bucket、reset credits；`account/usage/read` 返回 lifetime tokens、peak daily tokens、longest running turn、streak、daily buckets，并且要求 Codex services-backed auth，API-key-only 和 Bedrock auth 不支持。([OpenAI Developers][6])                                                                               | **later**；只可选做 read-only quota panel。                                                            | 可放 Device/Account detail pane，不放主对话。Near-term 最多显示 `rate-limited / degraded / unknown` badge。                                            | 需要真实 ChatGPT/Codex 服务认证、计划/工作区权限、付费或配额状态。必须防止把 plan/email/workspace 信息泄露到 shared Control Plane。                                               |

## 近期 stage 应该纳入什么

**建议 near-term 只纳入三类 projection，不纳入高级功能本体。**

| Web-facing projection | 来源能力                                                        | UI 支持面                                              | 说明                                                                                  |
| --------------------- | ----------------------------------------------------------- | --------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `device.auth`         | `account/read`，不是 `getAuthStatus`                           | Sidebar device badge、Status Strip、Right Detail Pane | 显示是否 signed in、是否 requires auth、provider、planType 的脱敏值。不要让 Web 持有 token。            |
| `device.platform`     | OS + Codex version + app-server maturity + transport health | Device detail / Status Strip                        | 显示 macOS/Windows/WSL、Codex version、app-server experimental、stdio/ws/unix transport。 |
| `device.sandbox`      | Windows host 上的 sandbox readiness/setup state               | 仅 Windows device detail / warning badge             | 只读状态优先；setup 需要用户明确确认，且由 Worker 执行。                                                 |

## 暂停条件清单

| 暂停条件                                                  | 影响能力                                                  | 为什么暂停                                                                                                                                                          |
| ----------------------------------------------------- | ----------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **真实 ChatGPT 登录、SSO、MFA、passkey**                     | account login、remote host pairing、rate/usage、realtime | 官方 remote mobile setup 要求同账号/工作区、host 醒着在线并已登录；部分场景需要 MFA/SSO/passkey。Codex cloud auth 也强调 MFA。([OpenAI Developers][2])                                        |
| **API key / access token / externally managed token** | `account/login/start`、`chatgptAuthTokens`、usage/rate  | API key 按 Platform 计费；`chatgptAuthTokens` 是 experimental，要求 host app 自己管理 token lifecycle 和 refresh。([OpenAI Developers][7])                                   |
| **Windows native 环境与管理员/本地策略**                        | `windowsSandbox/setupStart/readiness`                 | `elevated` sandbox 依赖低权限用户、ACL、防火墙、本地策略；`unelevated` 是 weaker fallback。([OpenAI Developers][5])                                                                |
| **外部网络暴露 app-server**                                 | 所有 app-server 能力，尤其 remote console                    | 官方 app-server WebSocket transport 是 experimental/unsupported；非 loopback WS 在 rollout 期间默认可能允许 unauthenticated connection，暴露前必须配置 auth。([OpenAI Developers][6]) |
| **音频权限、Realtime 服务、实时计费/额度**                          | realtime voice                                        | Realtime 是长连接 live audio session；Codex app-server realtime 仍是 experimental，并且事件不进入 durable thread history。([OpenAI Developers][3])                             |
| **日志/路径/命令输出/会话内容上传**                                 | `feedback/upload`                                     | feedback upload 天然会携带 logs、conversation id、extra log files；必须先有 redaction、preview、explicit consent、默认本地保存。([OpenAI Developers][6])                             |
| **本机配置写入与外部 agent 资产迁移**                              | `externalAgentConfig/import`                          | 该能力会迁移 AGENTS、config、skills、plugins、MCP、hooks、commands、sessions；应先实现 dry-run、backup、diff、rollback。([OpenAI Developers][6])                                     |

## 最终路线判断

1. **Near-term**：只做 `account/read` 的脱敏状态、device/platform/sandbox capability projection、rate-limit degraded badge 的最小只读状态。
2. **Later**：Windows sandbox setup wizard、rate/usage detail panel、external agent import wizard。
3. **Experimental-watch**：`thread/realtime/*`、`windowsSandbox/readiness`、`windows/worldWritableWarning`、`chatgptAuthTokens`。
4. **Not a Codex Remote target**：`feedback/upload` 作为核心功能不做；只可能作为后续 diagnostics。
5. **Insufficient evidence**：`getAuthStatus` 作为 public app-server API 不成立；当前应统一落到官方 `account/read`。`windows/worldWritableWarning` 目前只能作为 Windows Worker 内部 warning 投影，不应做成独立产品能力。
