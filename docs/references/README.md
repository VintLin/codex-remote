# References

This directory stores evidence and background. It is not a source of truth for product scope, API fields, package boundaries, or design tokens.

Use root documents for active decisions:

- `PRODUCT.md`: product positioning, MVP scope, users, non-goals, experience principles.
- `DESIGN.md`: visual system and frontend style.
- `PLAN.md`: roadmap, phase status, risks, and next action.
- `PROJECT_STRUCTURE.md`: directory ownership and dependency direction.
- `QUESTIONS.md`: open research queue and local verification backlog.

## Current References

- `codex-app-server.md`: explanatory app-server protocol reference. Generated protocol artifacts in `packages/codex-protocol` remain the type source of truth.
- `development-context.md`: adopted stage context and future-stage technical guardrails that are too specific for `AGENTS.md`.
- `openai-codex-app-pages/pages/`: official Codex App product behavior snapshots. These are UX and behavior references, not API contracts.
- `questions/SYNTHESIS.md`: research answer synthesis and adoption status.
- `research/参考项目架构调研报告 v0.2.md`: reference-project architecture evidence and adoption/rejection rationale.

## Archived Material

Absorbed PRDs, old research summaries, prompt logs, import metadata, and duplicated metadata snapshots live under `docs/archives/`.
